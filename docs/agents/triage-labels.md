# Triage 标签

工程技能使用五个规范化 triage 角色。下表将这些角色映射到本仓库问题跟踪器中的实际标签。

| 技能中的角色 | GitHub 标签 | 含义 |
| --- | --- | --- |
| `needs-triage` | `needs-triage` | 需要维护者评估该 Issue |
| `needs-info` | `needs-info` | 等待报告者补充信息 |
| `ready-for-agent` | `ready-for-agent` | 信息完整，可交给异步 Agent 执行 |
| `ready-for-human` | `ready-for-human` | 需要人工实现或处理 |
| `wontfix` | `wontfix` | 不会采取行动 |

当技能提及某个角色，例如“应用可由异步 Agent 执行的 triage 标签”时，应使用上表对应的 GitHub 标签。

如果仓库未来调整标签命名，只需修改表格右侧的 GitHub 标签列。
